## 进程，线程，协程的区别是什么？
总的来说，进程是操作系统进行资源分配的基本单位，而线程通常是 CPU 调度的基本单位，协程是一种**用户态**的轻量级线程。

进程是操作系统进行资源分配的基本单位，所谓**资源**，进程拥有独立的**虚拟地址空间，通常包括代码段、数据段、堆和栈**等区域。不同进程之间默认不共享内存，因此进程间通信需要通过特定机制实现，例如**管道、消息队列、共享内存、Socket**等。由于进程之间相互隔离，一个进程崩溃通常不会直接影响其他进程，因此稳定性和安全性相对较高。但进程切换的开销较大，涉及进程地址空间的切换，另外涉及不同进程的线程执行上下文的切换。现代操作系统中 CPU 实际调度的通常是线程，因此所谓进程切换，本质上通常是从一个进程中的线程切换到另一个进程中的线程。**CPU 上下文切换**是指保存当前线程的**程序计数器、寄存器、栈指针、状态寄存器**等执行现场，并恢复下一个线程的执行现场，使其能够从上次暂停的位置继续执行。

线程是进程内的一个执行流，也是现代操作系统中 CPU 调度和分派的基本单位。与进程不同，**同一进程内的多个线程共享进程的虚拟地址空间**，包括代码段、数据段、堆和全局变量等资源；但每个线程都有自己的**栈、程序计数器、寄存器状态**等执行上下文。线程之间通信更加高效，因为它们可以直接读写共享内存，但也因此需要使用锁、信号量、条件变量、原子操作等机制保证并发安全。
同一进程内的线程切换开销通常小于进程切换，因为它主要涉及线程 CPU 上下文的保存和恢复，例如程序计数器、寄存器、栈指针、状态寄存器等，一般不需要切换进程地址空间。
但这个**开销小是相对的**。对于这种直接由CPU调度的线程，线程切换过程通常由系统调用、中断或异常使 CPU 进入内核态，然后内核调度器保存当前线程上下文，选择下一个线程，恢复其上下文，最后返回用户态继续执行。

用户态和内核态之间的切换，以及内核线程的上下文切换，都会带来一定性能开销。为了更高效地处理大量并发任务，现代编程语言和运行时引入了协程等轻量级并发机制。例如，C++20 引入语言级协程，使函数可以挂起并在之后恢复；Go 提供 goroutine，由 Go runtime 负责调度；Java 在 JDK 21 中正式引入虚拟线程，由 JDK 将大量虚拟线程调度到较少的平台线程上执行。
**协程通常运行在用户态**，其切换不需要内核调度器直接参与，因此切换开销比线程更小。协程切换时主要保存和恢复协程自身的执行状态，例如**当前执行位置、局部变量、调用状态以及栈信息**等，而不需要进行完整的内核级线程上下文切换
